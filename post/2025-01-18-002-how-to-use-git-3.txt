f:T107b,<h1 id="git-사용법-원격-저장소-연결-워크플로우" tabindex="-1">Git 사용법: 원격 저장소 연결 &amp; 워크플로우</h1>
<h2 id="5-원격-저장소-연결" tabindex="-1">5. 원격 저장소 연결</h2>
<p>원격 저장소(Remote Repository)는 GitHub, GitLab 등과 같은 플랫폼에서 호스팅되는 저장소입니다. 원격 저장소를 활용하면 다른 사람과 협업하거나 데이터를 백업할 수 있습니다.</p>
<h3 id="원격-저장소-추가" tabindex="-1">원격 저장소 추가</h3>
<p>원격 저장소 URL을 추가합니다.</p>
<pre><code class="hljs language-bash">git remote add origin &lt;원격 저장소 URL&gt;
</code></pre>
<p>예시:</p>
<pre><code class="hljs language-bash">git remote add origin https://github.com/username/repository.git
</code></pre>
<h3 id="원격-저장소-확인" tabindex="-1">원격 저장소 확인</h3>
<p>현재 연결된 원격 저장소 목록을 확인합니다.</p>
<pre><code class="hljs language-bash">git remote -v
</code></pre>
<h3 id="변경-사항-푸시" tabindex="-1">변경 사항 푸시</h3>
<p>로컬 저장소의 변경 사항을 원격 저장소로 보냅니다.</p>
<pre><code class="hljs language-bash">git push -u origin &lt;브랜치명&gt;
</code></pre>
<p>예시:</p>
<pre><code class="hljs language-bash">git push -u origin main
</code></pre>
<h3 id="변경-사항-가져오기" tabindex="-1">변경 사항 가져오기</h3>
<p>원격 저장소의 최신 변경 사항을 로컬 저장소로 가져옵니다.</p>
<pre><code class="hljs language-bash">git pull origin &lt;브랜치명&gt;
</code></pre>
<h3 id="저장소-클론" tabindex="-1">저장소 클론</h3>
<p>원격 저장소를 복제합니다.</p>
<pre><code class="hljs language-bash">git <span class="hljs-built_in">clone</span> &lt;원격 저장소 URL&gt;
</code></pre>
<p>예시:</p>
<pre><code class="hljs language-bash">git <span class="hljs-built_in">clone</span> https://github.com/username/repository.git
</code></pre>
<hr>
<h2 id="6-git-워크플로우" tabindex="-1">6. Git 워크플로우</h2>
<p>Git 워크플로우는 팀이나 개인이 Git을 사용하는 방식을 정의한 것입니다. 올바른 워크플로우를 선택하면 효율적인 협업이 가능합니다.</p>
<h3 id="기본-워크플로우" tabindex="-1">기본 워크플로우</h3>
<ol>
<li>
<p><strong>로컬에서 작업 시작</strong>:</p>
<ul>
<li>브랜치를 생성하고 전환:<pre><code class="hljs language-bash">git branch &lt;브랜치명&gt;
git switch &lt;브랜치명&gt;
</code></pre>
</li>
<li>파일 변경 후 스테이징:<pre><code class="hljs language-bash">git add .
</code></pre>
</li>
<li>커밋:<pre><code class="hljs language-bash">git commit -m <span class="hljs-string">&quot;작업 내용&quot;</span>
</code></pre>
</li>
</ul>
</li>
<li>
<p><strong>원격 저장소로 푸시</strong>:</p>
<pre><code class="hljs language-bash">git push origin &lt;브랜치명&gt;
</code></pre>
</li>
<li>
<p><strong>변경 사항 병합</strong>:</p>
<ul>
<li>원격에서 Pull Request를 생성하거나, 로컬에서 브랜치를 병합:<pre><code class="hljs language-bash">git merge &lt;브랜치명&gt;
</code></pre>
</li>
</ul>
</li>
</ol>
<hr>
<h3 id="주요-git-워크플로우" tabindex="-1">주요 Git 워크플로우</h3>
<h4 id="1-feature-branch-workflow" tabindex="-1">1. <strong>Feature Branch Workflow</strong></h4>
<ul>
<li>새로운 기능을 작업할 때마다 별도의 브랜치를 생성하여 작업.</li>
<li>병합 전에 코드 리뷰 수행.</li>
</ul>
<h4 id="2-gitflow-workflow" tabindex="-1">2. <strong>GitFlow Workflow</strong></h4>
<ul>
<li>브랜치를 <code>main</code>, <code>develop</code>으로 나누고, 기능 브랜치와 릴리스 브랜치를 추가로 사용.</li>
<li>대규모 프로젝트에 적합.</li>
</ul>
<h4 id="3-forking-workflow" tabindex="-1">3. <strong>Forking Workflow</strong></h4>
<ul>
<li>원본 저장소를 포크한 뒤, 개인 저장소에서 작업 후 Pull Request를 통해 기여.</li>
<li>오픈소스 프로젝트에서 주로 사용.</li>
</ul>
<hr>
