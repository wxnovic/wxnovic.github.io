f:T1ade,<h1 id="git-사용법-파일-복구-및-되돌리기-협업-방법" tabindex="-1">Git 사용법: 파일 복구 및 되돌리기 &amp; 협업 방법</h1>
<h2 id="9-git-파일-복구-및-되돌리기" tabindex="-1">9. Git 파일 복구 및 되돌리기</h2>
<p>Git을 사용할 때 실수로 잘못된 변경을 커밋하거나 삭제했다면 걱정하지 않아도 됩니다. Git은 다양한 복구 및 되돌리기 도구를 제공하며, 이를 통해 손쉽게 변경 사항을 되돌릴 수 있습니다.</p>
<hr>
<h3 id="작업-디렉토리-변경사항-되돌리기" tabindex="-1"><strong>작업 디렉토리 변경사항 되돌리기</strong></h3>
<p>작업 중 수정한 파일을 이전 상태로 되돌립니다. (스테이징되지 않은 변경 사항만 되돌림)</p>
<pre><code class="hljs language-bash">git restore &lt;파일명&gt;
</code></pre>
<h4 id="모든-파일의-변경-사항-되돌리기" tabindex="-1">모든 파일의 변경 사항 되돌리기</h4>
<pre><code class="hljs language-bash">git restore .
</code></pre>
<blockquote>
<p><strong>참고</strong>: 되돌린 내용은 복구되지 않으므로 신중히 사용해야 합니다.</p>
</blockquote>
<hr>
<h3 id="스테이징된-변경사항-취소" tabindex="-1"><strong>스테이징된 변경사항 취소</strong></h3>
<p><code>git add</code>로 스테이징된 파일을 다시 스테이징 해제합니다.</p>
<pre><code class="hljs language-bash">git restore --staged &lt;파일명&gt;
</code></pre>
<hr>
<h3 id="커밋-되돌리기" tabindex="-1"><strong>커밋 되돌리기</strong></h3>
<h4 id="마지막-커밋-수정" tabindex="-1">마지막 커밋 수정</h4>
<p>마지막 커밋 메시지나 내용을 수정하려면 아래 명령어를 사용합니다.</p>
<pre><code class="hljs language-bash">git commit --amend
</code></pre>
<h4 id="특정-커밋-삭제-soft-reset" tabindex="-1">특정 커밋 삭제 (Soft Reset)</h4>
<p>커밋을 삭제하고 변경 사항은 유지합니다.</p>
<pre><code class="hljs language-bash">git reset --soft &lt;커밋 ID&gt;
</code></pre>
<h4 id="특정-커밋-삭제-hard-reset" tabindex="-1">특정 커밋 삭제 (Hard Reset)</h4>
<p>커밋을 삭제하고 변경 사항도 모두 제거합니다.</p>
<pre><code class="hljs language-bash">git reset --hard &lt;커밋 ID&gt;
</code></pre>
<blockquote>
<p><strong>주의</strong>: <code>--hard</code> 옵션은 되돌릴 수 없으므로 중요한 데이터를 잃지 않도록 주의하세요.</p>
</blockquote>
<hr>
<h3 id="커밋-되돌리기-revert" tabindex="-1"><strong>커밋 되돌리기 (Revert)</strong></h3>
<p>이전 커밋을 취소하면서 새로운 커밋으로 변경 사항을 기록합니다. 협업 환경에서 안전하게 사용됩니다.</p>
<pre><code class="hljs language-bash">git revert &lt;커밋 ID&gt;
</code></pre>
<hr>
<h3 id="git-stash-사용" tabindex="-1"><strong>Git Stash 사용</strong></h3>
<p>작업을 중단하고 다른 작업을 수행해야 할 때, 변경 사항을 임시 저장할 수 있습니다.</p>
<ol>
<li>변경 사항 임시 저장:<pre><code class="hljs language-bash">git stash
</code></pre>
</li>
<li>저장한 변경 사항 확인:<pre><code class="hljs language-bash">git stash list
</code></pre>
</li>
<li>저장한 변경 사항 복원:<pre><code class="hljs language-bash">git stash apply
</code></pre>
</li>
<li>저장한 변경 사항 삭제:<pre><code class="hljs language-bash">git stash drop
</code></pre>
</li>
</ol>
<hr>
<h2 id="10-git과-협업-방법" tabindex="-1">10. Git과 협업 방법</h2>
<p>Git은 팀 협업을 효과적으로 지원하는 도구입니다. 특히, 브랜치와 원격 저장소를 활용하면 효율적으로 작업을 분리하고 병합할 수 있습니다.</p>
<hr>
<h3 id="원격-저장소를-활용한-협업" tabindex="-1"><strong>원격 저장소를 활용한 협업</strong></h3>
<ol>
<li>
<p><strong>저장소 클론</strong>
팀원의 원격 저장소를 복제하여 협업을 시작합니다.</p>
<pre><code class="hljs language-bash">git <span class="hljs-built_in">clone</span> &lt;원격 저장소 URL&gt;
</code></pre>
</li>
<li>
<p><strong>변경 사항 푸시</strong>
로컬에서 작업한 내용을 원격 저장소로 보냅니다.</p>
<pre><code class="hljs language-bash">git push origin &lt;브랜치명&gt;
</code></pre>
</li>
<li>
<p><strong>변경 사항 가져오기</strong>
원격 저장소의 최신 변경 사항을 가져옵니다.</p>
<pre><code class="hljs language-bash">git pull origin &lt;브랜치명&gt;
</code></pre>
</li>
</ol>
<hr>
<h3 id="브랜치를-활용한-협업" tabindex="-1"><strong>브랜치를 활용한 협업</strong></h3>
<ol>
<li>
<p><strong>기능별 브랜치 생성</strong>
팀원들이 독립적으로 작업할 수 있도록 각자 브랜치를 만듭니다.</p>
<pre><code class="hljs language-bash">git branch &lt;브랜치명&gt;
git switch &lt;브랜치명&gt;
</code></pre>
</li>
<li>
<p><strong>Pull Request (PR)</strong>
작업이 완료되면 Pull Request를 생성해 코드 리뷰를 요청합니다. Pull Request는 협업에서 매우 중요한 도구로, 팀원이 코드 리뷰를 통해 품질을 확인하고 병합합니다.</p>
</li>
<li>
<p><strong>병합 (Merge)</strong>
리뷰가 완료된 브랜치를 메인 브랜치에 병합합니다.</p>
<pre><code class="hljs language-bash">git merge &lt;브랜치명&gt;
</code></pre>
</li>
</ol>
<hr>
<h3 id="충돌-해결" tabindex="-1"><strong>충돌 해결</strong></h3>
<p>병합 중 충돌이 발생하면 아래 절차를 따릅니다.</p>
<ol>
<li>충돌이 발생한 파일 수정.</li>
<li>수정한 파일을 스테이징.<pre><code class="hljs language-bash">git add &lt;파일명&gt;
</code></pre>
</li>
<li>변경 사항을 커밋.<pre><code class="hljs language-bash">git commit
</code></pre>
</li>
</ol>
<hr>
<h3 id="github와-협업" tabindex="-1"><strong>GitHub와 협업</strong></h3>
<p>GitHub를 활용하면 아래와 같은 추가 협업 기능을 사용할 수 있습니다.</p>
<ol>
<li><strong>Issues</strong>: 작업 항목을 관리하고 팀원들과 소통.</li>
<li><strong>Actions</strong>: CI/CD(자동화된 테스트 및 배포) 설정.</li>
<li><strong>Wiki</strong>: 프로젝트 문서를 공유.</li>
</ol>
<hr>
<h2 id="마무리" tabindex="-1">마무리</h2>
<p>Git은 협업과 복구 기능에서 탁월한 성능을 발휘하는 도구입니다. 특히, 커밋 되돌리기와 충돌 해결 방법을 숙지하면 협업 중 발생하는 문제를 효과적으로 해결할 수 있습니다. 또한, GitHub의 고급 기능을 활용하면 협업을 더 체계적으로 관리할 수 있습니다. 💻✨</p>
