Git 사용법: 기본 명령어와 브랜치 관리

3. Git 기본 명령어

Git에서 가장 자주 사용되는 기본 명령어들을 소개합니다.

저장소 초기화

새로운 Git 저장소를 생성합니다.

git init

파일 상태 확인

저장소에서 추적 중인 파일들의 상태를 확인합니다.

git status

파일 추가

파일을 스테이징 영역에 추가합니다.

git add <파일명>

모든 파일을 추가하려면:

git add .

변경 사항 커밋

변경 사항을 커밋합니다. 커밋 메시지를 포함해야 합니다.

git commit -m "커밋 메시지"

파일 삭제

Git에서 추적 중인 파일을 삭제합니다.

git rm <파일명>

4. Git 브랜치 관리

Git 브랜치는 독립적으로 작업할 수 있는 환경을 제공합니다. 새로운 기능을 개발하거나 실험할 때 유용합니다.

브랜치 확인

현재 브랜치를 확인하거나 모든 브랜치를 나열합니다.

git branch

브랜치 생성

새로운 브랜치를 만듭니다.

git branch <브랜치명>

브랜치 전환

다른 브랜치로 전환합니다.

git checkout <브랜치명>

또는 Git 2.23 이상 버전에서는:

git switch <브랜치명>

브랜치 병합

현재 브랜치에 다른 브랜치를 병합합니다.

git merge <브랜치명>

충돌 해결

병합 중 충돌이 발생하면 수정 후 다음 명령어로 충돌을 해결합니다.

git add <파일명>
git commit

브랜치 삭제

더 이상 필요 없는 브랜치를 삭제합니다.

git branch -d <브랜치명>

강제로 삭제하려면:

git branch -D <브랜치명>

이제 Git의 기본 명령어와 브랜치 관리를 활용해 보다 체계적으로 프로젝트를 관리할 수 있습니다! 🚀
