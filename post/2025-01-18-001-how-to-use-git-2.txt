f:Td29,<h1 id="git-사용법-기본-명령어와-브랜치-관리" tabindex="-1">Git 사용법: 기본 명령어와 브랜치 관리</h1>
<h2 id="3-git-기본-명령어" tabindex="-1">3. Git 기본 명령어</h2>
<p>Git에서 가장 자주 사용되는 기본 명령어들을 소개합니다.</p>
<h3 id="저장소-초기화" tabindex="-1">저장소 초기화</h3>
<p>새로운 Git 저장소를 생성합니다.</p>
<pre><code class="hljs language-bash">git init
</code></pre>
<h3 id="파일-상태-확인" tabindex="-1">파일 상태 확인</h3>
<p>저장소에서 추적 중인 파일들의 상태를 확인합니다.</p>
<pre><code class="hljs language-bash">git status
</code></pre>
<h3 id="파일-추가" tabindex="-1">파일 추가</h3>
<p>파일을 스테이징 영역에 추가합니다.</p>
<pre><code class="hljs language-bash">git add &lt;파일명&gt;
</code></pre>
<p>모든 파일을 추가하려면:</p>
<pre><code class="hljs language-bash">git add .
</code></pre>
<h3 id="변경-사항-커밋" tabindex="-1">변경 사항 커밋</h3>
<p>변경 사항을 커밋합니다. 커밋 메시지를 포함해야 합니다.</p>
<pre><code class="hljs language-bash">git commit -m <span class="hljs-string">&quot;커밋 메시지&quot;</span>
</code></pre>
<h3 id="파일-삭제" tabindex="-1">파일 삭제</h3>
<p>Git에서 추적 중인 파일을 삭제합니다.</p>
<pre><code class="hljs language-bash">git <span class="hljs-built_in">rm</span> &lt;파일명&gt;
</code></pre>
<hr>
<h2 id="4-git-브랜치-관리" tabindex="-1">4. Git 브랜치 관리</h2>
<p>Git 브랜치는 독립적으로 작업할 수 있는 환경을 제공합니다. 새로운 기능을 개발하거나 실험할 때 유용합니다.</p>
<h3 id="브랜치-확인" tabindex="-1">브랜치 확인</h3>
<p>현재 브랜치를 확인하거나 모든 브랜치를 나열합니다.</p>
<pre><code class="hljs language-bash">git branch
</code></pre>
<h3 id="브랜치-생성" tabindex="-1">브랜치 생성</h3>
<p>새로운 브랜치를 만듭니다.</p>
<pre><code class="hljs language-bash">git branch &lt;브랜치명&gt;
</code></pre>
<h3 id="브랜치-전환" tabindex="-1">브랜치 전환</h3>
<p>다른 브랜치로 전환합니다.</p>
<pre><code class="hljs language-bash">git checkout &lt;브랜치명&gt;
</code></pre>
<p>또는 Git 2.23 이상 버전에서는:</p>
<pre><code class="hljs language-bash">git switch &lt;브랜치명&gt;
</code></pre>
<h3 id="브랜치-병합" tabindex="-1">브랜치 병합</h3>
<p>현재 브랜치에 다른 브랜치를 병합합니다.</p>
<pre><code class="hljs language-bash">git merge &lt;브랜치명&gt;
</code></pre>
<h3 id="충돌-해결" tabindex="-1">충돌 해결</h3>
<p>병합 중 충돌이 발생하면 수정 후 다음 명령어로 충돌을 해결합니다.</p>
<pre><code class="hljs language-bash">git add &lt;파일명&gt;
git commit
</code></pre>
<h3 id="브랜치-삭제" tabindex="-1">브랜치 삭제</h3>
<p>더 이상 필요 없는 브랜치를 삭제합니다.</p>
<pre><code class="hljs language-bash">git branch -d &lt;브랜치명&gt;
</code></pre>
<p>강제로 삭제하려면:</p>
<pre><code class="hljs language-bash">git branch -D &lt;브랜치명&gt;
</code></pre>
<hr>
<p>이제 Git의 기본 명령어와 브랜치 관리를 활용해 보다 체계적으로 프로젝트를 관리할 수 있습니다! 🚀</p>
