f:T11b8,<h1 id="git-사용법-로그와-히스토리-관리-태그-사용법" tabindex="-1">Git 사용법: 로그와 히스토리 관리 &amp; 태그 사용법</h1>
<h2 id="7-git-로그와-히스토리-관리" tabindex="-1">7. Git 로그와 히스토리 관리</h2>
<p>Git은 파일 변경 이력을 철저히 관리합니다. 적절히 로그와 히스토리를 활용하면 특정 시점의 코드 상태를 이해하고, 문제 해결에 필요한 정보를 빠르게 찾을 수 있습니다.</p>
<h3 id="커밋-히스토리-확인" tabindex="-1">커밋 히스토리 확인</h3>
<p>Git의 모든 커밋 이력을 확인하려면 아래 명령어를 사용하세요.</p>
<pre><code class="hljs language-bash">git <span class="hljs-built_in">log</span>
</code></pre>
<h4 id="옵션을-활용한-로그-출력" tabindex="-1">옵션을 활용한 로그 출력</h4>
<ul>
<li><strong>간단히 보기</strong>: 커밋 메시지와 ID를 한 줄로 표시.<pre><code class="hljs language-bash">git <span class="hljs-built_in">log</span> --oneline
</code></pre>
</li>
<li><strong>그래프 형태로 보기</strong>: 브랜치와 병합 내역을 시각적으로 확인.<pre><code class="hljs language-bash">git <span class="hljs-built_in">log</span> --graph
</code></pre>
</li>
<li><strong>특정 파일의 변경 기록 확인</strong>:<pre><code class="hljs language-bash">git <span class="hljs-built_in">log</span> -- &lt;파일명&gt;
</code></pre>
</li>
</ul>
<h3 id="특정-커밋-확인" tabindex="-1">특정 커밋 확인</h3>
<p>특정 커밋의 상세 내용을 확인할 때는 아래 명령어를 사용합니다.</p>
<pre><code class="hljs language-bash">git show &lt;커밋 ID&gt;
</code></pre>
<p>이 명령어는 해당 커밋에서 수정된 파일과 변경 내용을 보여줍니다.</p>
<hr>
<h2 id="8-git-태그-사용법" tabindex="-1">8. Git 태그 사용법</h2>
<p>태그(Tag)는 프로젝트에서 특정 시점을 표시하거나, 중요한 상태(예: 릴리스 버전)를 기록할 때 유용합니다. 일반적으로 소프트웨어 버전 관리에 많이 사용됩니다.</p>
<h3 id="태그의-종류" tabindex="-1">태그의 종류</h3>
<ol>
<li><strong>Annotated Tag (주석 태그)</strong>: 메타데이터와 서명을 포함하여 저장.</li>
<li><strong>Lightweight Tag (경량 태그)</strong>: 단순히 커밋을 가리키는 포인터 역할.</li>
</ol>
<h3 id="태그-생성" tabindex="-1">태그 생성</h3>
<ul>
<li>
<p><strong>Annotated Tag 생성</strong>:</p>
<pre><code class="hljs language-bash">git tag -a &lt;태그명&gt; -m <span class="hljs-string">&quot;태그 설명&quot;</span>
</code></pre>
<p>예시:</p>
<pre><code class="hljs language-bash">git tag -a v1.0 -m <span class="hljs-string">&quot;첫 번째 릴리스&quot;</span>
</code></pre>
</li>
<li>
<p><strong>Lightweight Tag 생성</strong>:</p>
<pre><code class="hljs language-bash">git tag &lt;태그명&gt;
</code></pre>
<p>예시:</p>
<pre><code class="hljs language-bash">git tag v1.0
</code></pre>
</li>
</ul>
<h3 id="태그-확인" tabindex="-1">태그 확인</h3>
<p>저장소의 모든 태그를 나열합니다.</p>
<pre><code class="hljs language-bash">git tag
</code></pre>
<h3 id="태그-푸시" tabindex="-1">태그 푸시</h3>
<p>원격 저장소에 태그를 푸시하려면 아래 명령어를 사용합니다.</p>
<pre><code class="hljs language-bash">git push origin &lt;태그명&gt;
</code></pre>
<p>모든 태그를 푸시하려면:</p>
<pre><code class="hljs language-bash">git push origin --tags
</code></pre>
<h3 id="태그-삭제" tabindex="-1">태그 삭제</h3>
<ul>
<li>
<p><strong>로컬 태그 삭제</strong>:</p>
<pre><code class="hljs language-bash">git tag -d &lt;태그명&gt;
</code></pre>
</li>
<li>
<p><strong>원격 태그 삭제</strong>:</p>
<pre><code class="hljs language-bash">git push origin --delete &lt;태그명&gt;
</code></pre>
</li>
</ul>
<hr>
<h2 id="마무리" tabindex="-1">마무리</h2>
<p>Git의 <strong>로그와 히스토리 관리</strong>를 활용하면 프로젝트의 전체 흐름을 정확히 이해할 수 있고, <strong>태그</strong>를 적절히 사용하면 중요한 시점을 명확히 관리할 수 있습니다. 특히 팀 협업 환경에서는 이러한 기능이 코드 안정성과 효율성을 높이는 데 큰 기여를 합니다. 🎯</p>
