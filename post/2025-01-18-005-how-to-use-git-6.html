Git 사용법: 고급 기능 & 유용한 팁

11. Git 고급 기능

Git의 기본 명령어를 충분히 익혔다면, 더 나아가 고급 기능을 활용해 작업 효율성과 협업 품질을 높일 수 있습니다. 여기서는 서브모듈, 리베이스, 스태시 등 고급 기능에 대해 자세히 설명합니다.

서브모듈(Submodule)

Git 서브모듈은 하나의 Git 저장소 내부에 다른 Git 저장소를 포함시키는 기능입니다. 이를 통해 프로젝트의 특정 디렉토리를 독립적인 버전 관리 대상으로 설정할 수 있습니다.

서브모듈 추가

git submodule add <서브모듈 URL> <디렉토리 경로>

예시:

git submodule add https://github.com/example/library.git external/library

서브모듈 초기화

저장소를 클론한 후 서브모듈을 초기화하고 업데이트합니다.

git submodule init
git submodule update

서브모듈 삭제

서브모듈 디렉토리와 관련 설정을 삭제합니다.

git submodule deinit -f <디렉토리>
rm -rf <디렉토리>
git rm <디렉토리>

리베이스(Rebase)

리베이스는 브랜치 병합의 대안으로, 브랜치를 다른 브랜치의 최신 상태 위로 재정렬합니다. 이를 통해 깔끔한 커밋 히스토리를 유지할 수 있습니다.

리베이스 실행

git rebase <브랜치명>

예시:

git rebase main

리베이스 중 충돌 해결

  1. 충돌 발생 시 파일 수정.
  2. 수정한 파일을 스테이징:
    git add <파일명>
  3. 리베이스 재개:
    git rebase --continue

주의: 리베이스는 협업 중인 브랜치에서 사용 시 주의가 필요합니다. 커밋 히스토리가 변경될 수 있기 때문입니다.

Git 스태시(Stash) 활용

Git 스태시는 작업 도중 변경 사항을 임시로 저장하고, 작업 환경을 초기화한 뒤 다시 복원할 수 있는 기능입니다.

변경 사항 임시 저장

git stash

특정 이름으로 저장

git stash push -m "작업 중인 UI 업데이트"

저장된 목록 보기

git stash list

변경 사항 복원

git stash apply

가장 최근 저장 내용 삭제

git stash drop

모든 스태시 삭제

git stash clear

12. Git 사용 팁

Git을 더 효율적으로 사용하기 위한 몇 가지 팁과 도구를 소개합니다.

.gitignore 파일 작성법

.gitignore 파일은 Git이 특정 파일이나 디렉토리를 무시하도록 설정하는 파일입니다. 예를 들어, 빌드 파일, 비밀번호가 포함된 설정 파일 등을 제외할 때 사용됩니다.

.gitignore 생성

echo "node_modules/" >> .gitignore
echo ".env" >> .gitignore

자주 사용되는 예제

# 환경설정 파일
.env

# 빌드 폴더
/dist
/build

# 로그 파일
*.log

# 운영체제 생성 파일
.DS_Store
Thumbs.db

.gitignore는 프로젝트 루트 디렉토리에 위치해야 합니다.

Git Alias 설정

자주 사용하는 명령어를 짧게 줄여서 사용할 수 있도록 별칭을 설정합니다.

Alias 설정

git config --global alias.st status
git config --global alias.cm "commit -m"
git config --global alias.co checkout
git config --global alias.br branch

설정 확인

git config --list

유용한 Git 도구

Git 사용을 더 쉽고 시각적으로 도와주는 GUI 도구와 플러그인을 활용해 보세요.

  1. GitKraken

    • 사용하기 쉬운 인터페이스와 강력한 브랜치 관리 기능 제공.
    • 다운로드: https://www.gitkraken.com/

  2. Sourcetree

    • Atlassian에서 제공하는 무료 Git GUI 클라이언트.
    • 다운로드: https://www.sourcetreeapp.com/

  3. Visual Studio Code Git 플러그인

    • VS Code에서 내장된 Git 도구로 커밋, 병합, 푸시를 간단하게 처리 가능.
    • 설치: https://code.visualstudio.com/

Git 명령어 자동 완성

터미널에서 Git 명령어를 자동 완성하도록 설정하면 생산성이 높아집니다.

Bash 사용자

Git의 자동 완성 스크립트를 다운로드하고 .bashrc에 추가:

curl -o ~/.git-completion.bash https://raw.githubusercontent.com/git/git/master/contrib/completion/git-completion.bash
echo "source ~/.git-completion.bash" >> ~/.bashrc
source ~/.bashrc

Zsh 사용자

Oh My Zsh의 Git 플러그인을 사용하여 자동 완성을 설정:

plugins=(git)

마무리

Git의 고급 기능과 팁은 복잡한 프로젝트에서도 깔끔한 작업 환경을 유지하고 협업 효율을 극대화합니다. 서브모듈로 외부 라이브러리를 관리하거나 리베이스로 히스토리를 정리하고, .gitignore를 통해 민감한 파일을 보호하는 습관을 들이면 더욱 생산적인 Git 사용이 가능합니다. 🚀
