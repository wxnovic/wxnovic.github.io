Git 사용법: 원격 저장소 연결 & 워크플로우

5. 원격 저장소 연결

원격 저장소(Remote Repository)는 GitHub, GitLab 등과 같은 플랫폼에서 호스팅되는 저장소입니다. 원격 저장소를 활용하면 다른 사람과 협업하거나 데이터를 백업할 수 있습니다.

원격 저장소 추가

원격 저장소 URL을 추가합니다.

git remote add origin <원격 저장소 URL>

예시:

git remote add origin https://github.com/username/repository.git

원격 저장소 확인

현재 연결된 원격 저장소 목록을 확인합니다.

git remote -v

변경 사항 푸시

로컬 저장소의 변경 사항을 원격 저장소로 보냅니다.

git push -u origin <브랜치명>

예시:

git push -u origin main

변경 사항 가져오기

원격 저장소의 최신 변경 사항을 로컬 저장소로 가져옵니다.

git pull origin <브랜치명>

저장소 클론

원격 저장소를 복제합니다.

git clone <원격 저장소 URL>

예시:

git clone https://github.com/username/repository.git

6. Git 워크플로우

Git 워크플로우는 팀이나 개인이 Git을 사용하는 방식을 정의한 것입니다. 올바른 워크플로우를 선택하면 효율적인 협업이 가능합니다.

기본 워크플로우

  1. 로컬에서 작업 시작:

    • 브랜치를 생성하고 전환:
      git branch <브랜치명>
git switch <브랜치명>
    • 파일 변경 후 스테이징:
      git add .
    • 커밋:
      git commit -m "작업 내용"

  2. 원격 저장소로 푸시:

    git push origin <브랜치명>

  3. 변경 사항 병합:

    • 원격에서 Pull Request를 생성하거나, 로컬에서 브랜치를 병합:
      git merge <브랜치명>

주요 Git 워크플로우

1. Feature Branch Workflow

  • 새로운 기능을 작업할 때마다 별도의 브랜치를 생성하여 작업.
  • 병합 전에 코드 리뷰 수행.

2. GitFlow Workflow

  • 브랜치를 main, develop으로 나누고, 기능 브랜치와 릴리스 브랜치를 추가로 사용.
  • 대규모 프로젝트에 적합.

3. Forking Workflow

  • 원본 저장소를 포크한 뒤, 개인 저장소에서 작업 후 Pull Request를 통해 기여.
  • 오픈소스 프로젝트에서 주로 사용.

Git의 워크플로우를 이해하고 활용하면 협업 및 작업 효율성이 크게 향상됩니다! 🎯
